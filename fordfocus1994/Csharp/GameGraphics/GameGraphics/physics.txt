Машинный код - самый низкий уровень языка программирования.
Языки ассемблера - низкий уровень языка программирования.
Паскаль - язык программирования высокого уровня.
Языки сверхвысокого уровня программирования - указывается, что нужно сделать, а не как.

Трансляция - преобразование кода из одного языка в другой, более низкого уровня.
Компиляция - преобразование языка высокого уровня в язык низкого уровня, близкий к машинному коду.
Интерпретация - моделирование виртуальной машины для работы с конкретным высокоуровневым языком без трансляции.

Cвязывание - процесс установления соответствия между объектами и их свойствами в программе на формальном языке (операции, операторы, данные) и элементами архитектуры компьютера (команды, адреса).
Время связывания - фаза подготовки программы к выполнению (трансляция, компоновка, загрузка), на которой производится это действие. 

///////////////////////////////////////Пратт и Зелковиц

Объект данных - один или несколько однотипных элементов данных, объединенных в одно целое в виртуальном компьютере в некоторый момент выполнения программы. Именнованная область памяти, где хранятся данные.
	Связанная переменная: объект данных, связанный со значением х.
	Определяемые программистом или системой ОД.
	Атрибуты определяют тип данных, количество и тип значений, которые могут содержаться в ОД, логическую организацию значений.
	Значение данных - отдельное число, символ или указатель на другой объект данных. Конкретная комбинация битов в памяти компьютера.
	Переменная: объект данных, явно определенный и названный программистом.
	Константа: переменная, имя которой связано со значением на протяжении всего времени жизни программы. Информация о константе известна уже во время трансляции.
	Связывание:
		Тип - этот атрибут ассоциирует объект данных со множеством значений, которые могут содержаться в этом объекте данных.
		Местоположение - связывание ОД с определенной областью памяти.
		Значение - операция присваивания.
		Имя - связывание ОД с одним или несколькими именами, по которым можно обратиться к ОД.
		Компонент - связь ОД с одним или более ОД, компонентом которых он является (значение указателя).

Типы данных - это некоторый класс объектов данных вместе с набором операций для создания и работы с ними. Программа работает с конкретными ОД, язык программирования имеет дело с типами данных.
	Примитивные типы, встроенные в язык. Есть возможность объявлять новые типы.
	Элементы спецификации типа данных:
		Атрибуты, которые характеризуют ОД заданного типа (имя, тип). 
			Некоторые атрибуты могут храниться в дескрипторе во время работы программы - в массиве предварительной информации. 
			Другие могут использоваться только для определения местоположения ОД в памяти компьютера.
			Значение атрибута ОД отличается от значения, которое содержится в этом ОД.
		Значения, которые могут принимать ОД заданного типа.
			Тип ОД определяет множество допустимых значений, которые могут содержаться в этом оД.
			Упорядоченное множество - множество значений, допустимых для элементарных ОД.
		Операции, которые определяют возможные манипуляции над объектами заданного типа.
			Элементарные операции - часть определения языка.
			Могут определяться программистом в виде подпрограмм или методов как часть определения класса.
			Каждому входному аргументу сопостовляется однозначно определенный результат.
			Область определения операции (множество допустимых значений аргументов) и область значений (множество значений для результатов).
			Действие операции определяет, каковы будут результаты  для данного набора аргументов.
			Операции задаются алгоритмами.
			Определяются в аппаратной части, как подпрограммы или последовательность кодов.
	Атрибуты ОД обычно представляются синтаксически через объявления или определения типов.
	Значения представляются литералами или именнованными константами.
	Вызов операции может происходить при помощи специальных символов, встроенных процедур или функций.
	Элементы реализации:
		Способ представления ОД этого типа в памяти компьютера в процессе выполнения программы. 
			Обычно не зависит от местоположения, описывается в терминах необходимого размера блока памяти и компоновки атрибутов и значений данных в пределе этого блока.
			Адрес первого слова или байта обычно представляет местоположение ОД.
		Способ представления операций, определенных для этого типа данных, через конкретные алгоритмы и процедуры, которые используются для манипуляцицй с выбранной формой представления ОД в памяти.
		Способ реализации элементарных типов данных определяется аппаратной частью компьютера: элементарные операции типа данных можно реализовывать при помощи 
			операций, определенных с помощью аппаратной части. Иначе их придется моделироавть с помощью подпрограмм.
		Атрибуты могут определяться компилятором или содержаться в дескрипторе.
	Элементарный тип данных может содержать только одно значение.	

Объявление - оператор программы, который сообщает транслятору информацию об именах и типах ОД. 
	Размещение объявлений задает время жизни ОД. Объявление связывает ОД с именами на период времени их жизни.
	Явные объявления и неявные. В объявлении также можно задать значение ОД.
	Объявление операций: задать тип аргументов и результата в прототипе.
	Функции объявлений:
		Выбор способа представления в памяти.
		Улучшение управления памятью. В С объекты, объявленные в начале подпрограммы, имеют время жизни, равное времени жизни подпрограммы, и помещаются в единый блок памяти.
		Полиморфные функции. Перегруженные операции, обозначающие общие операции.

Контроль типов - проверка того, что каждая операция получает правильное количество аргументов правильного типа.
	Динамический контроль типов - во время выполнения программы. Статический - во время компиляции. 
	дескриптор указывает на принадлежность ОД к определенному типу.
	Сильная типизация - отсутствие ошибок определения типа - статический контроль.

Приведение типов (неявное преобразование).

Присваивание - операция изменения связывания ОД со значением. 

Инициализация присвоение значения ОД. Создание ОД - выделение блока памяти для ОД с имеющейся на момент выделения комбинацией битов. 

Скалярные типы данных - только один атрибут (значение). Элементарные типы данных.
Численные типы данных - целые и вещественные числа.
	Дескриптор отсутствует, хранится в отдельном или в том же слове.
	Поддиапазоны. 
	Перечисление - упорядочен последовательность различных элементов. В памяти представлены короткими целыми числами. 
Логический тип данных - истина и ложь.
Символьный тип данных - единственный символ. 
Строки символов.
Указатели - хранят ссылку на местоположение ОД.


Файл - структура данных, располагающаяся на внешнем устройстве хранения, обладающая большим временем жизни.
	Состоят из ОД определенной длины. Для ввода и вывода обычно представляются в текстовом виде.
	Указатель текущей позиции в файле.
	Режими чтения и записи. Запись - указатель всегда в конце, присваивание этой позиции нового компонента. Чтение - доступ к компоненту, следующему за позицией указателя.
	Открытие, чтение, запись, проверка конца, закрытие, проверка существования.
	Интерактивного ввода-вывода, прямого доступа, текстовые. 
Типизированный файл - файл, содержащий однотипные данные.
	
	
Структура данных - совокупность ОД, другие ОД - ее элементы.
	Однородная/неоднородная.
	Организация объектов - простая зависимость.
	Может меняться только значение компонента.
	
Список - упорядоченная последовательность структур данных, которые могут находиться в разных частях памяти, но при этом предыдущий элемент связан со следующим.

Массив - структура, определенное число элементов, линейная последовательность - вектор.
	Количество компонентов.
	Тип данных для каждого компонента.
	Список значений индексов, используемых для выбора компонентов.
	Индексация - операция выбора компонента.

Запись - структура данных, состоящая из фиксированного количества компонентов различных типов.
	Компоненты обозначаются идентификаторами.
	Компоненты в памяти расположены последовательно.

	
Синтаксис языка - система языковых категорий, относящихся к соединениям слов и строению предложений.
	Алфавит: набор символов, используемых в языке. 8 байт для представления символов.
	Идентификаторы: имена элементов языка.
	Символы операций: непосредственно символы операций или идентификаторы функций, их заменяющих.
	Ключевые слова: зарезервированные, фиксированные идентификаторы.
	Разделители: обозначает начало и конец синтаксического элемента.
	Выражения: функции, обрабатывающие данные и возвращающие значения.
	Операторы: инструкция/команда, исполняющая код, работающая с символами операций, идентификаторами, выражениями.
Семантика языка - смысловое значение элементов языка.
